# Go 学习路线

## 第 1 阶段：快速过语法

使用 `A Tour of Go` 快速熟悉：

- 基本语法
- 函数和控制流
- struct、slice、map
- method、interface
- goroutine、channel

目标不是全背下来，而是对 Go 的语言外形建立第一印象。

## 第 2 阶段：用书补理解

使用《The Go Programming Language》建立更扎实的理解。

要求：

- 边读边写最小例子
- 不只看懂，要能脱离原文自己写出同类代码
- 对 package、接口、并发、错误处理重点投入

## 第 3 阶段：完整跟一个项目

选一个难度适中的项目，连续几天完整跟完。

要求：

- 每天有明确输入和输出
- 不只跟敲，要理解每个模块为什么这样拆
- 跟完之后能说清数据流和调用关系

## 第 4 阶段：主动改项目

在已有项目基础上做小改动。

例如：

- 加缓存过期
- 加超时控制
- 加中间件
- 补日志和错误包装

这一阶段的重点是把“能看懂”推进到“能改动”。

## 第 5 阶段：独立重写简化版

独立实现一个简化版本，保留核心结构，删除次要细节。

这样做的意义：

- 检验是否真正掌握核心设计
- 暴露理解断层
- 训练从零组织代码的能力

## 第 6 阶段：补工程基础

最后补 Go 工程中必须掌握的基础习惯：

- `testing`
- `go mod`
- `context`
- goroutine 泄漏排查
- `pprof` 基础

这部分不一定最早学，但一定不能省略。
