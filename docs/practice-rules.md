# 练习规则

## 规则 1：不要只看懂

看懂示例不代表会写。

每学完一个知识点，至少做一次脱离资料的手写复现。

## 规则 2：不要只跟敲

跟着教程写只能证明你看得懂当前步骤，不能证明你能独立完成。

每次练习后都要补一个自己的小改动。

## 规则 3：优先最小闭环

如果一个概念没有理解清楚，先写最小可运行例子验证，不要直接在复杂项目里硬顶。

## 规则 4：写完要复盘

每次完成后，至少回答三个问题：

- 这个模块解决什么问题
- 核心数据结构是什么
- 如果让我重写，我会怎么拆

## 规则 5：复杂度要慢慢加

先写能跑的，再写清楚的，再写更合理的。

不要在入门阶段过早追求“高级架构”。
