# projects

这个目录用于存放项目跟练、功能扩展和独立重写版本。

## 目标

- 把 Go 放进真实代码，而不是停留在语法练习
- 训练包组织、接口抽象、错误处理、并发控制这些工程能力
- 通过功能改动验证自己是否真的理解项目结构

## 推荐练法

每次项目练习都分成三个阶段：

1. 跟着做，跑通主流程
2. 自己加一个小改动
3. 独立重写一个简化版

## 每日改动示例

- 给缓存加过期时间
- 给 Web 框架加中间件
- 给 RPC 加超时控制
- 给任务队列加重试
- 给接口层补日志或错误包装

## 建议结构

可以按项目名称拆目录，例如：

- `cache/`
- `web/`
- `rpc/`
- `rewrite/`

每个项目目录建议包含：

- `README.md`：项目目标和进度
- `notes.md`：关键理解和踩坑记录
- 源码目录
